## 一、产品定位与核心价值

### 1.1 产品定位

**CCEM Desktop** 是 Claude Code Environment Manager 的桌面应用版本，为**重度并行开发者**提供可视化的环境管理和会话控制能力。

### 1.2 核心用户场景

目标用户同时运行 **8 个 Claude Code 终端**（2 个四分屏），核心痛点是：

- 多窗口时难以快速定位目标会话
- 频繁切换环境和权限模式
- 需要了解 Token 使用量和成本，获得成就感

### 1.3 桌面版的不可替代价值

| CLI 能做 | 桌面版的增量价值 |
|---------|-----------------|
| `ccem use xxx` 切换环境 | **可视化状态**：一眼看到当前环境 |
| `ccem yolo` 切换权限 | **零记忆切换**：点击即切换，无需记命令 |
| 无 | **多会话管理**：Focus / 最小化 / 4分屏布局 |
| 解析 JSONL 日志 | **成就感系统**：Token 趋势图 + 成本统计 + 热力图 |

### 1.4 产品形态

- **主窗口**：完整的 6 Tab 管理界面
- **托盘菜单**：快捷操作入口（环境切换、权限切换、启动、会话 Focus）

## 二、Tab 架构与职责边界

### 2.1 六 Tab 架构

```
┌────────────────────────────────────────────────────────────────┐
│  [🏠 Home]  [💬 Sessions]  [🌐 Environments]  [📊 Analytics]   │
│  [✦ Skills]  [⚙️ Settings]                                    │
└────────────────────────────────────────────────────────────────┘
```

### 2.2 各 Tab 职责定义

| Tab | 核心职责 | ✅ 应该做 | ❌ 不应该做 |
|-----|---------|----------|------------|
| **🏠 Home** | 快速启动中心 | 启动按钮、当前状态、最近项目 | 详细统计、完整会话列表 |
| **💬 Sessions** | 多会话管理 | Focus、最小化、终止、4分屏布局 | 环境配置、Token 详情 |
| **🌐 Environments** | 环境+权限配置 | 环境 CRUD、权限切换入口 | 会话管理、统计数据 |
| **📊 Analytics** | 成本意识+成就感 | Token/费用趋势、连续天数、热力图 | 游戏化徽章、排行榜 |
| **✦ Skills** | 技能扩展（未来） | 已安装列表、添加/删除 | 重新设计技能系统 |
| **⚙️ Settings** | 应用配置 | 主题、启动选项（≤5项） | 设置膨胀 |

### 2.3 关键设计决策

1. **删除独立的 Permissions 页面** —— 权限切换融合到 Environments Tab
2. **权限模式不按环境绑定** —— 只有用户级别默认权限 + 启动时临时选择
3. **Home 是动作中心** —— 详情留给专属 Tab，避免信息仪表盘化
4. **Skills 复用 CLI** —— 调用 `ccem skill` 命令，不重新设计

- **目录**：上次使用的目录（如果仍存在）

### 4.3 会话卡片信息层级

| 优先级 | 信息 | 样式 |
|-------|------|------|
| **第一眼** | 项目名 | 16px 粗体 |
| **第二眼** | 环境 · 权限 | 14px 标签样式 |

- 🟢 **运行中**：有活动的会话
- 🟡 **空闲**：5 分钟无活动

| 操作 | 按钮 | 说明 |

| **终止** | `[✕]` | 终止会话进程 |
| **4分屏** | `[⊞]` | 将选中的 4 个会话排列成四分屏（未来功能） |

### 4.6 视图切换

- **卡片视图**：适合 ≤8 个会话，信息更丰富
- **列表视图**：适合 >8 个会话，信息更紧凑

```
└─────────────────────────────────────────────────────────────────┘
```

### 5.3 权限模式设计（关键决策）

| 层级 | 说明 | 持久性 |
|-----|------|-------|
| **用户级别默认权限** | 在 Settings 或此处设置 | 永久保存 |
| **临时权限** | 快速切换按钮，或 Home 页下拉选择 | 仅下次启动生效 |

**不支持按环境绑定权限** —— 权限是启动时的选择，与环境解耦。

### 5.4 权限模式说明

| 模式 | 说明 |
|-----|------|
| `yolo` | 完全开放，无限制 |
| `dev` | 标准开发权限，保护敏感文件 |
| `safe` | 保守权限，适合不熟悉的代码库 |
| `readonly` | 只读访问，禁止修改 |
| `ci` | CI/CD 流水线权限 |
| `audit` | 安全审计，只读+搜索 |

### 5.5 环境操作

| 操作 | 说明 |
|-----|------|
| **添加** | 弹出对话框，支持从预设选择（GLM/Kimi/DeepSeek等） |
| **编辑** | 修改 Base URL、Model、API Key |
| **切换** | 设为当前环境，立即生效 |
| **删除** | 需二次确认，official 受保护不可删除 |

### 6.3 图表详细设计

#### Token 消耗分布图

| 维度 | 说明 |
|-----|------|
| **时间粒度** | 小时 / 日 / 周 / 月 切换 |
| **图表类型** | 折线图（带层叠）/ 柱状图 切换 |
| **数据分组** | 按环境分色（Official 蓝、GLM 绿、DeepSeek 紫等） |
| **层叠模式** | 折线图显示各环境叠加趋势 |

#### 模型分布

| 元素 | 说明 |
|-----|------|
| **横向条形图** | 按使用占比排序 |
| **右侧显示费用** | 让用户直观看到各模型成本 |

#### 活跃热力图

| 元素 | 说明 |
|-----|------|
| **GitHub 风格日历** | 按日显示 Token 使用量 |
| **颜色深浅** | 表示当日使用量级别 |
| **Hover 显示** | 具体日期 + Token 数 + 费用 |

### 6.4 成就感系统边界

| ✅ 保留 | ❌ 拒绝 |
|--------|--------|
| 趋势箭头（↑18%） | 徽章收集系统 |
| 连续使用天数 | 用户排行榜 |
| 里程碑提示（Toast） | 经验值/等级 |
| 热力图视觉满足 | 每日签到奖励 |

### 6.5 里程碑设计

- **Token 里程碑**：100K → 500K → 1M → 5M → 10M
- **费用里程碑**：$10 → $50 → $100 → $500
- **连续使用**：7天 → 30天 → 100天 → 365天
- **节省成本**：$10 → $50 → $100（对比官方价格）

达成里程碑时显示 Toast 通知，不打断工作流。

|-----|------|

## 十一、设计总结

### 11.1 六轮 PK 的核心收获

1. **用户场景驱动设计** —— "8 个四分屏"改变了整个讨论方向
2. **边界清晰是生命线** —— 每个 Tab 必须有明确的"做什么"和"不做什么"
3. **成就感 ≠ 游戏化** —— 趋势箭头和连续天数足够，不需要徽章系统
4. **Home 是动作中心** —— 不是信息仪表盘，详情留给专属 Tab
5. **复用 CLI 是工程智慧** —— Skills 不需要重新设计

### 11.2 矛盾论的产品设计应用

- **Designer 的贡献**：完整思考了每个场景的设计方案
- **Stormer 的贡献**：确保每个功能都有存在的理由
- **综合的智慧**：该有的都有，不该有的一个不留
